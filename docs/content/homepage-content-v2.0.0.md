# Homepage Content — v2.0.0

## Overview

This document defines the content structure and messaging for the Home page of the personal website `v2.0.0`.

The goal of this version is to reposition the website from a general personal/technical portfolio into a professional services and portfolio website focused on:

> Sound · AV Networking · Technical Infrastructure

The Home page must communicate quickly:

- Who I am.
- What technical territory I work in.
- What services I offer.
- Why my hybrid profile is valuable.
- What kind of projects/case studies support this positioning.
- How a client or collaborator can contact me.

The Home page should not try to explain everything. Its purpose is to guide the visitor toward Services, Portfolio/Projects and Contact.

---

# 1. Home Page Structure

The proposed Home page structure for `v2.0.0` is:

1. Hero
2. Services Preview
3. Hybrid Profile
4. Featured Case Studies
5. Call To Action

Optional supporting section:

6. Technical Proof / This website is part of the system

This optional section can be kept if it supports the portfolio narrative, but it should not dominate the Home page.

# 9. What should not dominate the Home page

The Home page should avoid becoming too dense or too internally focused.

Avoid making the Home mainly about:

- A personal biography.
- A long explanation of DevOps.
- A full technical CV.
- A list of every tool known.
- A philosophical vision statement.
- Too many future roles not yet offered.
- Too many secondary services.

Those contents can live in:

- About
- Vision
- Projects
- Services
- Blog / future content
