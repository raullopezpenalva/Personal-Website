# Services Definition — v2.0.0

## Overview

This document defines the professional services offered through the personal website `v2.0.0`.

The goal of this version is to reposition the website as a services and portfolio platform focused on:

> Sound Engineering · AV Networking · Technical Systems

The services are based on a hybrid technical profile that combines real audiovisual field experience, sound engineering, networking, systems thinking, documentation, troubleshooting and infrastructure mindset.

The website should not communicate a generic “sound technician” profile or a generic “junior DevOps” profile.

The core positioning is:

> Sound engineer specialized in AV systems, networking and audio over IP for live events and production environments.

Or, in a more compact form:

> Sound Engineer / AV Systems / Event Networking

---

# 1. Service Categories

The main service categories for `v2.0.0` are:

1. Sound Engineering
2. Monitor Engineering
3. Sound System Engineering
4. AV Networking for Events
5. Technical Planning & Consulting

These services can be offered independently or combined depending on the project.

The commercial goal is to communicate that the service is not only about operating equipment, but about understanding the full technical system around an event, venue or production environment.

---

# 2. Sound Engineering

## Short description

Sound engineering support for live events, concerts, audiovisual productions and professional production environments.

## Positioning

This service covers the operation and technical control of sound during an event, with a focus on reliability, communication, preparation and musical criteria.

It is not only about mixing audio. It is about understanding the show, the signal flow, the technical context and the pressure of live production.

## What this service includes

- FOH mixing.
- Soundcheck support.
- Input list and stage plot review.
- Patch verification.
- Console preparation when possible.
- Show file preparation when possible.
- Signal flow troubleshooting.
- Communication with production, artists and technical teams.
- Basic coordination with monitor, stage and system teams.
- Post-show notes and technical feedback when useful.

## Typical use cases

- Live concerts.
- Music events.
- Corporate events with live sound.
- Small and medium festivals.
- Cultural venues.
- Events that need a reliable sound engineer with system awareness.

## Value delivered

- Clear and controlled sound for the audience.
- Professional preparation before the event.
- Faster troubleshooting during setup and showtime.
- Better coordination with the technical team.
- Reduced technical uncertainty for production.

## Related skills and tools

- Digital consoles.
- FOH workflows.
- Signal routing.
- EQ, dynamics and effects.
- Dante Controller when audio over IP is involved.
- Documentation and show files.
- Technical checklists.

# 4. Sound System Engineering

## Short description

Sound system design, verification, measurement and optimization for live events, venues and production environments.

## Positioning

This service focuses on the sound system as a complete technical system: coverage, alignment, coherence, subs, fills, delays and response across the audience area.

It connects sound engineering with a deeper systems mindset.

## What this service includes

- PA system verification.
- Basic system design review.
- Coverage analysis.
- Speaker placement review.
- Subwoofer strategy review.
- Delay and fill alignment.
- Measurement-based tuning.
- System EQ support.
- SPL and coverage checks.
- Technical documentation and post-event notes when required.

## Typical use cases

- Concerts.
- Festivals.
- Venues.
- Events with distributed sound systems.
- Productions using subs, fills or delay systems.
- Events where PA consistency and coverage matter.

## Value delivered

- More consistent sound coverage.
- Better alignment between PA, subs, fills and delays.
- Objective technical decision-making through measurement.
- Better support for FOH engineers.
- Reduced risk of system-related sound problems.

## Related skills and tools

- Smaart.
- Measurement microphones.
- Audio interfaces.
- PA prediction tools.
- ArrayCalc.
- Soundvision.
- MAPP.
- R1.
- Dante Controller.
- System documentation.
- Technical reports.

## Important scope note

This service should be offered progressively and with clearly defined scope.

Depending on the project, it may include:

- System verification.
- System optimization.
- Measurement support.
- Technical supervision.
- Documentation.

It should not imply full responsibility for a system that has been designed, supplied or installed by third parties unless the scope is explicitly agreed.

# 5. AV Networking for Events

## Short description

Design, deployment, configuration and troubleshooting of IP networks for audiovisual events and production systems.

## Positioning

This is one of the most differential services in the portfolio.

Many AV environments depend on networks for audio, video, control, lighting, intercom, streaming and production workflows. This service focuses on making those networks clear, stable and documented.

The core idea:

> Understanding the event as infrastructure.

## What this service includes

- Event network planning.
- Basic network topology design.
- IP addressing plans.
- DHCP or static IP planning.
- VLAN planning when needed.
- Managed switch configuration.
- Technical Wi-Fi setup.
- Dante network support.
- NDI / video over IP support.
- Art-Net / sACN awareness.
- Streaming network support.
- Connectivity troubleshooting.
- Network documentation.
- Port and cable labelling when required.
- Network report after the event when useful.

## Typical use cases

- Dante audio networks.
- NDI workflows.
- Streaming setups.
- Production control networks.
- Technical Wi-Fi for control devices.
- Events with consoles, tablets, computers, cameras or servers on the same network.
- Venues that need documented AV network infrastructure.
- Small teams that need someone who understands both AV and IP networking.

## Value delivered

- More stable technical networks.
- Fewer IP conflicts and connectivity issues.
- Better separation between systems.
- Clearer documentation for production teams.
- Faster troubleshooting.
- Stronger connection between audio, video, control and infrastructure.

## Related skills and tools

- Switching.
- VLANs.
- DHCP.
- Static IP planning.
- Subnetting.
- Multicast basics.
- Dante Controller.
- Dante Virtual Soundcard.
- NDI Tools.
- Wireshark.
- iperf3.
- nmap.
- arp-scan.
- Linux CLI tools.
- UniFi Network.
- Network Scout.
- Network documentation templates.

## Important scope note

This service must always define what is included and what is not included.

Possible included scope:

- Network design.
- Switch configuration.
- Event deployment.
- Troubleshooting.
- Documentation.
- Monitoring during the event.

Possible exclusions:

- Unlimited support.
- Responsibility for undocumented third-party systems.
- Repair of damaged cabling unless agreed.
- 24/7 availability.
- Full IT support for non-event systems.

# 6. Technical Planning & Consulting

## Short description

Technical planning, documentation and pre-production support for events, venues and audiovisual projects.

## Positioning

This service exists to prevent technical improvisation.

It is aimed at clients, venues or production teams that need someone to review the technical needs before the event, detect risks, coordinate requirements and create a clear plan.

This is especially important when different providers, artists or technical teams are involved.

## What this service includes

- Technical requirement review.
- Rider review.
- Input list review.
- Stage plot review.
- AV system needs analysis.
- Network needs analysis.
- Provider coordination.
- Risk detection.
- Technical documentation.
- Pre-production checklists.
- Site visit when required.
- Technical recommendations.
- Scope definition.
- Post-event improvement notes when useful.

## Typical use cases

- Events with unclear technical requirements.
- Festivals.
- Venues receiving external productions.
- Clients hiring several AV providers.
- Projects where sound, video, network and production need coordination.
- Events where the client wants technical supervision without assuming unnecessary risk.

## Value delivered

- Less improvisation.
- Better communication between client, production and providers.
- Clearer technical responsibilities.
- Earlier detection of problems.
- Better documentation.
- More professional event execution.

## Related skills and tools

- Technical documentation.
- Production checklists.
- Rider analysis.
- Signal flow diagrams.
- Network diagrams.
- Patch sheets.
- Input lists.
- System reports.
- Obsidian.
- Spreadsheets.
- PDF documentation.
- Project planning.

## Important scope note

This service is especially useful for protecting professional boundaries.

If a client requests only on-site supervision without previous planning, the scope must clearly state that responsibility is limited to what can be verified during the agreed service window.

The service should separate:

- Planning.
- Supervision.
- Operation.
- Responsibility over third-party systems.

- AV systems understanding.
- Networking knowledge.

- Documentation.

> I understand the event as a complete technical system.
